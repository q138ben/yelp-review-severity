The stakeholder & the pain

Priya can't read 10,000 reviews a week.

VP of Operations, a ~300-location casual-dining group. Thousands of feedback items land weekly — Yelp/Google, in-app, comment cards, support tickets, social. Three problems:

volume

Nobody reads it all

The signal that a location is going bad arrives late — or never.

lagging

Stars trail reality

By the time the public average visibly drops, the lost customers are already gone. Most channels carry no star at all.

no "why"

Which lever is failing?

A bad rating doesn't say whether #217 is slow service, cold food, or a dirty room — three different fixes.

Jobs to be done:  triage (route the worst to the right manager in minutes)  ·  diagnose (which aspect, per location)  ·  early-warn (catch the slide before the public average tanks). Rating is causally tied to revenue — this is retention, not reporting.
The pain

Why this is a modelling problem

Not "read the stars" — impute severity from text.

star-less channels

~13% carry no rating

Tips (908,915) + photo captions (~97k) ≈ 1M items with no star. Every channel an operator adds grows the gap. Value = a consistent severity signal from text alone.

the mixed middle

A lexicon collapses

Word-level polarity sums to mush on "great food but an hour wait" — exactly where judgement is needed. The rating is ordered, not categorical.

asymmetric cost

1★-as-5★ ≫ 3★-as-2★

Frame as ordinal regression; judge on rank-agreement (QWK, Spearman) that penalises far misses. Accuracy would reward "always 5★" — the corpus is 46% 5★.

Everything downstream is built to make the narrative actionable — score it, decompose it, and flag when the model itself is unsure.

Framing

The build, mapped to her three jobs

One encoder. Three jobs.

triage · who to call first

Ordinal encoder

Fine-tuned DistilBERT → calibrated 1–5 severity from text. QWK 0.879 on 20,179 held-out reviews.

diagnose · which lever

Same encoder, decomposed

Route clauses to aspects, score each with the same encoder → per-aspect 1–5 + the drivers.

qa · is this off?

Disagreement flag

Where the model strongly disagrees with the given star → mixed / mislabelled cases for a human.

One model, two views. The encoder scores the whole review (best QWK — use it to rank); the same encoder, run per aspect, decomposes that score (use it to act). The breakdown is a faithful decomposition of the ranking score — not a second model.

Notebook 01 · EDA — data quality & the split

Trust the data, then build a leakage-safe wall.

scope

99.9% English

langdetect on a 30k random sample → scope the encoder + tokenizer to English; the rest is negligible and out of scope.

integrity

Keys pristine

0 duplicate review / business IDs. Duplicate content is tiny — ~0.18% dup reviews, ~0.09% dup listings → no dedup pass needed.

completeness

Reviews near-complete

Business-side hours/attributes are ~9–15% gappy; categories (0.07% missing) is safe to scope aspects on.

Leakage-safe split — by business, not by row.
  • Each business is hashed deterministically to train / val / test (70 / 15 / 15) — every review of a location stays on one side of the wall.
  • Verified: 0 businesses span splits, and the three stay representative on both stars and length.
  • Near-oracle features — a user's or a business's historical average star (≈0.5 correlation with the row's own rating) — are excluded from the model's features: triage must work on new text at new locations, not memorise a history.

Notebook 02 · Ordinal model — approach & robust training

Stars are ordered — so is the loss, and so is the metric.

Model ladder — all on the same leakage-safe test split

  • Trivial — predict the training mean (a floor).
  • TF-IDF + Ridge — a strong classical text baseline.
  • DistilBERT-SST2 + regression head — a sentiment-pretrained encoder, single output, MSE on the float star.

Metrics = MAE/RMSE plus rank-agreement (Spearman, QWK). Two reviewers give 2★ for opposite reasons — absolute error alone hides whether the model preserves the ordering of severity.

robust fine-tuning — not a fixed 2-epoch pass
  • Gradient clipping (max-norm 1.0) — a fresh head on a pretrained trunk spikes gradients.
  • Per-epoch monitoring — train/val loss, val QWK, grad-norm logged every epoch.
  • Early stop + best-epoch restore — up to 100 epochs; predict with the best weights, not the last.
What the run showed: the fresh head exploded the gradient in epoch 1 (pre-clip norm ≈ 120, loss ≈ 5.4) — clipping absorbed it. Train loss then fell to ~0.06 while val loss stalled and the train/val gap widened — overfitting. Early stopping restored the best epoch (8, by val QWK); the last epoch trailed the peak by ≈ 0.002 QWK, so best-epoch restore takes the peak for free.

Architecture

Production — governed, versioned, rollback-able

The notebook and the cloud run the same loop.

  • Parity by construction. Training lives in src/ordinal.train_ordinal — imported by both the notebook and train.py; config.yaml is the single source of hyperparameters, guarded by a parity test.
  • QWK gate (0.80). A model below the bar can never reach production.
  • Immutable versions. Each run writes models/<timestamp> → a distinct artifact → a new, rollback-able Cloud Run revision. No mutable "latest".
  • Rollback. Redeploy an older version (deploy-only branch), or shift traffic to a retained revision instantly.

Governance

In operation — closing the loop

Flag what's off, route the rest.

the third job · QA

Disagreement flag

The encoder's score against the given star — the widest gaps surface mislabels and genuinely mixed reviews (a 1★ full of praise; a 5★ that lists problems) for a human. This closes the triage · diagnose · QA trilogy the overview promised.

confidence-banded routing

Auto-act, or escalate

Auto-act on confident extremes; send the ambiguous 2–3★ middle and long 800+ reviews to human review — exactly where the model is least sure and severity concentrates.

reliability

Trustworthy ranking

QWK 0.879; in a 10-review live spot-check, 9/10 within one star and every error a near-miss. The severity ordering — who gets attention first — is dependable.

guardrails

Human-in-the-loop, never punitive

A prioritisation aid, not a verdict. Drift (a COVID-era dip) → monitored retrain. English-scoped. Academic Yelp licence → a demonstration, not a commercial deployment.
